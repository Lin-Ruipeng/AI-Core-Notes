### 第 6 课（对应大纲第 4 课）：解线性方程组的数值稳定性 —— 消灭显式求逆
既然你学过数值分析，那我们就跳过“如何手搓高斯消元法”这种大二教材内容。在真实的 HPC 与端侧 AI 工程中，**没有任何人会手写 LU 分解，更没有任何人会去算矩阵的逆。**
本节课，我们要面对的是数值线性代数中最核心的“戒律”，以及它为什么在传感器标定中是致命的。
#### 1. 理论拆解：IMU 标定与法方程的“毒性”
**传感器场景映射**：
在做 IMU 标定（比如标定加速度计的零偏和标度因数）时，我们通常会采集大量数据，建立超定方程组 $Ax = b$（方程个数远大于未知数 $x$ 的个数）。
为了求解，数学上会转化为**正规方程**：
$$A^T A x = A^T b$$
这里的 $A^T A$ 是一个对称正定矩阵。看起来很完美？**大错特错！**
数值分析告诉我们，矩阵 $A^T A$ 的条件数是 $A$ 的条件数的**平方**。如果原始观测矩阵 $A$ 稍微有点病态（条件数 $10^4$），$A^T A$ 的条件数就会飙到 $10^8$。这意味着在计算机里，你的有效数字会瞬间损失 8 位！
#### 2. HPC 绝对禁忌：显式求逆
面对 $Ax=b$，刚学完线性代数的人最喜欢写：`x = A.inverse() * b`。
在 HPC 领域，这是会被 Code Review 直接打回并骂娘的写法。原因有两点：
*   **数值灾难（精度杀手）**：求逆的过程，本质上是解 $n$ 个列向量构成的方程组。计算量巨大，中间产生的舍入误差会成倍累积。算出逆矩阵 $A^{-1}$ 后，再乘以 $b$，又引入了一次矩阵乘法的误差。两步操作，把浮点数的有效数字啃得干干净净。
*   **性能地板（速度杀手）**：求逆的计算复杂度是 $O(n^3)$。而后续的矩阵乘向量 $A^{-1}b$ 又是 $O(n^2)$。总共约 $\frac{2}{3}n^3 + n^2$ 次浮点运算。而直接分解求解，只需 $\frac{1}{3}n^3$ 次。**显式求逆比直接求解慢了整整一倍！**
#### 3. HPC 杀手锏：Cholesky 分解与 `.solve()`
针对 $Ax=b$（特别是 $A$ 是对称正定矩阵时），工业界唯一认可的做法是使用 **Cholesky 分解**（将 $A$ 分解为下三角矩阵 $L$ 和其转置 $L^T$ 的乘积，$A = LL^T$）。
在 C++ 中（通常使用 Eigen 库），你绝对不能写 `x = A.inverse() * b`，而是必须调用求解器接口：
`x = A.llt().solve(b);`
它的底层逻辑是：拿到 $b$ 后，先解 $Ly = b$（前向替换），再解 $L^T x = y$（后向替换）。全程没有求逆，没有多余的乘法，精度达到理论极限，速度最快。

### 📝 课后实战作业（引入工业级数学库 Eigen）
在真实的 C++ HPC 开发中，我们不会用 `std::vector<std::vector<double>>` 来存矩阵，而是使用业界标杆库 **Eigen**。它是一个纯头文件库，编译极快，性能与手写汇编媲美。
**课前准备（极度重要）**：
你需要安装 Eigen 库。
*   **Mac**: `brew install eigen`
*   **Ubuntu/Debian**: `sudo apt install libeigen3-dev`
*   *(注意：Eigen 是 header-only 的，不需要链接，只要 include 路径对即可。通常安装在 `/usr/include/eigen3`)*
**题目：希尔伯特矩阵的处刑 —— 对比求逆与求解**
希尔伯特矩阵 $H$ 是数值分析界著名的“毒药”，它的元素 $H_{ij} = \frac{1}{i+j-1}$。随着矩阵维度 $N$ 增加，它的条件数呈指数级爆炸，任何常规的浮点运算在它面前都会原形毕露。
请使用 C++ 完成以下任务：
1.  **引入 Eigen**：
    ```cpp
    #include <Eigen/Dense>
    ```
2.  **构造毒药**：构造一个 $10 \times 10$ 的希尔伯特矩阵 $H$，以及一个精确解全为 `1.0` 的向量 $x_{true}$。计算出理论上的 $b = H \times x_{true}$。
3.  **方法 A（禁忌之法）**：使用 Eigen 的 `.inverse()` 显式求出 $H^{-1}$，然后计算 $x_A = H^{-1} \times b$。
4.  **方法 B（HPC 正道）**：利用 $H$ 是对称正定矩阵的特性，使用 Cholesky 分解求解：`x_B = H.llt().solve(b);`。
5.  **方法 C（暴力破解）**：如果不利用对称性，使用普通的 LU 分解（部分主元消去）：`x_C = H.lu().solve(b);`。
6.  **验证与报告**：
    *   计算 $x_A, x_B, x_C$ 与真实解 $x_{true}$ 的误差范数（使用 Eigen 的 `.norm()` 方法，它计算的是 L2 范数，即均方根误差）。
    *   打印这三种误差。
7.  **编译命令**：
    ```bash
    g++ main.cpp -O3 -march=native -I /usr/include/eigen3
    ```
    *(如果提示找不到头文件，请用 `find / -name "Eigen" 2>/dev/null` 找到你机器上的 Eigen 路径，替换 `-I` 后面的路径)*
