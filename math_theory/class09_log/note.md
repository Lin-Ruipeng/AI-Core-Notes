### 第 9 课（对应大纲第 7 课）：对数概率空间 —— 拯救被乘法抹杀的世界
在模块三的开头，我们学会了用 C++ 生成高斯噪声。今天，我们要面对这些噪声聚集在一起时，对计算机造成的最大绝望。
#### 1. 理论拆解：浮点数的“乘法黑洞”
**传感器场景映射**：
假设你写了一个算法，试图通过最大似然估计（MLE）来标定 IMU 的零偏。你让设备静止了 1 秒，采样率 1000Hz，得到了 1000 个加速度计观测值 $z_1, z_2, ..., z_{1000}$。
根据概率论，这 1000 个独立同分布观测值的联合概率（似然函数）是它们各自概率密度函数（PDF）的**连乘**：
$$P(Z|X) = \prod_{i=1}^{1000} P(z_i|X)$$
对于一维标准正态分布，单个点的概率密度最大值（在均值处）约为 $0.3989$。
你在 C++ 里做连乘：
$0.3989 \times 0.3989 \times 0.3989 \dots$
乘到第 1000 次时，结果是多少？
答案是 **`0.0`**。
在 C++ 中，`double` 的最小正规浮点数大约是 $2.2 \times 10^{-308}$。只要连乘的结果小于这个值，就会发生**下溢出**，直接变成绝对的 0。一旦变成 0，后续无论你乘以什么，哪怕是无穷大，结果永远是 0。你的似然函数彻底死亡，无法进行任何梯度下降或求导。
#### 2. HPC 杀手锏：对数似然
数学家给出的解药极其简单：**取对数**。
根据对数的性质 $\log(AB) = \log(A) + \log(B)$，连乘变成了连加：
$$\log P(Z|X) = \sum_{i=1}^{1000} \log P(z_i|X)$$
**为什么加法能拯救世界？**
因为 `double` 的表数范围是对称的。它能表示小到 $10^{-308}$ 的数，就能表示小到 $-10^{308}$ 的负数。即使概率极小，取对数后只是一个绝对值很大的负数（比如 `-50000.23`），在计算机里依然是精确的！加法永远只会让这个负数越来越大，绝不会下溢出变成 0。
#### 3. 工程级多维高斯对数似然推导（HPC 必备）
在真实的 3D 空间中，观测值 $z$ 是向量，协方差是 $3 \times 3$ 矩阵 $\Sigma$。直接套公式算 $\log P(z)$ 会有巨大的性能陷阱：
$$\log P(z) = -\frac{1}{2} \left[ D \log(2\pi) + \log|\Sigma| + (z-\mu)^T \Sigma^{-1} (z-\mu) \right]$$
（$D$ 是维度，这里是 3）
**性能陷阱与解法**：
*   **陷阱 1：算 $\Sigma^{-1}$**。绝对禁止！用上一课的 `.llt().solve()` 解线性系统。
*   **陷阱 2：算行列式 $\log|\Sigma|$**。如果直接调用 `Sigma.determinant()`，底层还是算 LU 分解求行列式，慢且容易数值溢出。
*   **HPC 正道：利用 Cholesky 分解的副产物**。
    因为 $\Sigma = L L^T$，所以行列式 $|\Sigma| = |L| \times |L^T| = |L|^2$（因为三角形矩阵的行列式等于对角线元素的乘积）。
    所以：**$\log|\Sigma| = 2 \times \log(|L|)$**。
    在 Eigen 中，拿到 $L$ 后，直接取其对角线元素，连乘再取 `log` 即可，速度极快且数值极其稳定。

### 📝 课后实战作业：见证下溢出与对数空间的奇迹
请继续使用 C++ 和 Eigen。这次我们需要结合上一课的随机数生成器。
**任务要求**：
1.  **构造噪声发生器**：
    定义一个 3 维的协方差矩阵 $\Sigma$，假设三个轴的方差都是 $0.01$（标准差 $0.1$），相互独立：
    ```cpp
    Eigen::MatrixXd Sigma(3, 3);
    Sigma << 0.01, 0, 0,
              0, 0.01, 0,
              0, 0, 0.01;
    ```
    均值向量 $\mu$ 设为全 0。
2.  **生成 1000 个带噪数据点**：
    使用上一课学到的多线程安全方法（单线程也可，重点是生成数据），生成 $N=1000$ 个 $3 \times 1$ 的向量，存储在 `std::vector<Eigen::Vector3d>` 中。
    *(提示：Eigen 可以直接与 `std::normal_distribution` 配合，如果协方差是对角的，直接对每个维度生成方差为 0.01 的随机数即可；如果想装逼，可以用 Eigen 的 `Eigen::MatrixXd::NullaryExpr` 结合随机数发生器生成协方差相关的噪声，但为了简化，对角阵直接按维度生成就行)*。
3.  **方法 A（物理毁灭：直接连乘）**：
    写一个函数 `double calc_direct_pdf(Vector3d z)` 计算单个点的概率密度（使用常量 `M_PI`）。
    用一个 `double likelihood = 1.0;` 在循环里不断 `likelihood *= calc_direct_pdf(z_i);`。
    打印最终的 `likelihood`。
4.  **方法 B（HPC 真经：对数似然与 Cholesky）**：
    *   在循环外，先对 $\Sigma$ 进行一次 `.llt()` 分解，拿到 $L$。
    *   利用 $L$ 的对角线元素，计算出常量部分：`log_det = 2.0 * log(L.diagonal().prod())` 以及常数项 `-1.5 * log(2.0 * M_PI) - 0.5 * log_det`。
    *   在循环内，对于每个 $z_i$，**不要去算逆矩阵**，而是用 `L.solve(z_i)` 求解 $L^{-1} z_i$（因为 $\mu=0$），然后算其范数的平方 `(L.solve(z_i)).squaredNorm()`。
    *   累加对数值：`log_likelihood += 常数项 - 0.5 * 马氏距离`。
    *   打印最终的 `log_likelihood`。为了证明对数空间的正确性，最后用 `exp(log_likelihood)` 打印一下还原出来的值，看看是不是和方法 A 一样变成了 0。

### 1. $P(z_i|X)$ 是上节课生成随机数的公式吗？
**绝对不是。它们是完全不同的两码事，只是描述的是同一个“高斯分布”。**
*   **上节课的公式（Box-Muller 变换）**：那是一个**“采样算法”**。就像是一台“造币机”，你给它喂进去均匀分布的废铁（$U_1, U_2$），它通过三角函数加工，吐出来符合高斯分布的硬币（随机数 $z$）。
*   **这节课的 $P(z_i|X)$（概率密度函数 PDF）**：这是一个**“评分函数”**。它不负责生成数字，它负责**评估**一个已经存在的数字 $z_i$ 出现的“可能性”有多大。
**打个比方**：
假设你在玩飞镖，靶心是均值 $\mu$。
上节课的代码是：**“闭上眼睛，按照高斯分布的概率把飞镖扔出去”**（生成 $z_i$）。
这节课的 $P(z_i|X)$ 是：**“你朋友扔了一镖，落在坐标 $z_i$ 处，你拿个放大镜去查标准正态分布表，算出他扔到这个特定位置的概率得分是多少”**。
在我们即将写的代码中，`calc_direct_pdf` 这个函数，**接收一个已有的飞镖坐标 $z_i$，返回一个概率得分（一个介于 0 到 0.4 之间的小数）**。

### 2. 为什么取对数后，前面会多出一个 $-\frac{1}{2}$？
这个 $-\frac{1}{2}$ 不是取对数变出来的，它是**高斯分布自己的基因里自带的**。我们用最简单的一维标准正态分布（均值0，方差1）来推导，你一眼就能看懂。
**步骤 A：写出原始的“评分公式”（一维高斯 PDF）**
你在高中或大学学过，一维正态分布长这样：
$$P(z) = \frac{1}{\sqrt{2\pi}} \exp\left( -\frac{z^2}{2} \right)$$
你看，指数部分本身就有一个 $\frac{1}{2}$，前面根号 $\sqrt{2\pi}$ 算出来也是 $2\pi$ 的 $\frac{1}{2}$ 次方。
**步骤 B：两边取对数（$\log$）**
对上面这个式子取 $\log$：
$$\log P(z) = \log\left( \frac{1}{\sqrt{2\pi}} \right) + \log\left( \exp\left( -\frac{z^2}{2} \right) \right)$$
根据对数性质化简：
1.  $\log(\frac{1}{A}) = -\log(A)$，所以前面变成 $-\log(\sqrt{2\pi}) = -\frac{1}{2}\log(2\pi)$
2.  $\log(\exp(x)) = x$，所以后面直接把指数拿下来，变成 $-\frac{z^2}{2} = -\frac{1}{2} z^2$
拼起来：
$$\log P(z) = -\frac{1}{2}\log(2\pi) - \frac{1}{2} z^2$$
**看出玄机了吗？** 把 $-\frac{1}{2}$ 提出来：
$$\log P(z) = -\frac{1}{2} \left[ \log(2\pi) + z^2 \right]$$
**步骤 C：推广到多维（3D IMU 传感器）**
在 3D 空间里，标量 $z$ 变成了向量 $z$，方差 $\sigma^2$ 变成了协方差矩阵 $\Sigma$。多维的原始公式是：
$$P(z) = \frac{1}{\sqrt{(2\pi)^D |\Sigma|}} \exp\left( -\frac{1}{2} (z-\mu)^T \Sigma^{-1} (z-\mu) \right)$$
（其中 $D=3$ 是维度，$|\Sigma|$ 是矩阵行列式，后面那一大坨叫“马氏距离”）
你按照刚才一模一样的套路取对数：
*   前面系数取对数：变成 $-\frac{D}{2}\log(2\pi) - \frac{1}{2}\log|\Sigma|$
*   后面指数拿下来：变成 $-\frac{1}{2} (z-\mu)^T \Sigma^{-1} (z-\mu)$
把这三项的 $-\frac{1}{2}$ 统统提取出来，就得到了我课上给你的终极公式：
$$\log P(z) = -\frac{1}{2} \left[ \underbrace{D\log(2\pi)}_{\text{常数项}} + \underbrace{\log|\Sigma|}_{\text{行列式项}} + \underbrace{(z-\mu)^T \Sigma^{-1} (z-\mu)}_{\text{马氏距离项}} \right]$$
### 总结
你作业里的 `calc_direct_pdf` 函数，就是要实现**步骤 C 中那个没有取对数的原始多维公式**（包含行列式和矩阵求逆）。
而方法 B，就是实现**步骤 C 中取完对数后的公式**。
