### 过渡课：最小二乘 —— 褪去概率外衣的赤裸代价
恭喜你走到了这里。在模块三的结尾，你看着屏幕上那个 `-11231` 的对数似然值。作为工程师，你的直觉会告诉你：“我不关心这个负数到底是多少，我只想知道**哪一个状态 X，能让这个负数的绝对值最小**！”
这就引出了整个 HPC 和状态估计领域的绝对霸主——**优化问题**。
#### 1. 理论拆解：最大似然是如何“堕落”成最小二乘的？
上一节课，我们推导出了神圣的负对数似然公式（为了简化，假设观测是一维的）：
$$-\log P(Z|X) = \frac{1}{2} \sum_{i=1}^{N} \left[ \log(2\pi\sigma^2) + \frac{(z_i - h_i(X))^2}{\sigma^2} \right]$$
在真实的 C++ 工程中，如果每次迭代都带着 $\log(2\pi\sigma^2)$ 去算，那是纯纯的浪费 CPU 周期。我们做一个**极其强硬的物理假设**：
**假设传感器的噪声方差 $\sigma^2$ 是个恒定的常数**（也就是噪声不随时间、不随状态改变，典型的高斯白噪声）。
既然是常数，那 $\log(2\pi\sigma^2)$ 就在对 $X$ 求导时变成了 0。$\frac{1}{2\sigma^2}$ 是个常数系数，提出来也不影响寻找最小值的位置。
所以，寻找最大似然估计 $X$，等价于寻找下面这个极其干净的式子的最小值：
$$\min_X \sum_{i=1}^{N} (z_i - h_i(X))^2$$
**看明白了吗？最小二乘法根本不是什么经验公式，它就是高斯白噪声假设下，最大似然估计把常数项扔掉后的“穷光蛋”版本！**
*   $z_i$：传感器的观测值
*   $h_i(X)$：根据当前猜测的状态 $X$，通过数学模型算出来的“预测值”
*   $e_i = z_i - h_i(X)$：**残差**。也就是“现实”与“理想”的差距。
*   整个优化的目的，就是**让所有残差的平方和最小**。
#### 2. HPC 性能陷阱：拒绝 sqrt 和多余的常数
在 C++ 中写代价函数，有一条铁律：**绝对不要出现 `sqrt`！**
假设你想算欧式距离的误差：
错误写法：`double cost = sqrt(dx*dx + dy*dy) - z;`
正确写法：`double cost = (dx*dx + dy*dy) - z*z;` （如果是距离平方的预测）
或者更常见的是：直接把 $e_i^2$ 拆开，写成矩阵形式 $e_i^T e_i$（即 Eigen 里的 `.squaredNorm()`）。
开方操作在 CPU 层面需要消耗数十个时钟周期，而且它是单调函数，不影响最小值的位置。在动辄迭代几万次的优化循环里，省掉 `sqrt` 能省下一大笔电费。

### 📝 课后实战作业：搭建非线性最小二乘问题的骨架
在接下来的课程中，我们要自己手写一个高斯牛顿法或 LM 算法。但在那之前，你必须先把“评价体系”（代价函数）写好。
**场景设定**：2D 激光雷达测距。
假设在一个 2D 平面上，有一个未知的目标点 $X = [x, y]^T$。
平面上有两个已知的雷达基站 $A$ 和 $B$。
*   雷达 A 的坐标是 `[0, 0]`
*   雷达 B 的坐标是 `[10, 0]`
真实的目标点在 `[3, 4]`。
所以理论上的距离：A 到目标是 5，B 到目标是 $\sqrt{(10-3)^2 + 4^2} \approx 8.06$。
现在你获得了带有高斯噪声的观测数据：测得 A 的距离是 `5.1`，测得 B 的距离是 `8.2`。
**任务要求**：
1.  **定义数据结构**：
    用一个简单的 `struct` 或者直接用变量，存下雷达基站的位置 $(x_{radar}, y_{radar})$ 和 观测到的距离 $z$。
2.  **编写代价函数**：
    写一个函数 `double ComputeCost(const Eigen::Vector2d& X, ...)`。
    输入是你猜测的目标坐标 $X$。
    内部逻辑：
    *   根据 $X$ 和雷达坐标，算出预测距离的平方：$h_{sq} = (X_x - x_{radar})^2 + (X_y - y_{radar})^2$
    *   把观测距离也平方：$z_{sq} = z^2$
    *   算单个残差：$e = h_{sq} - z_{sq}$ （注意：用距离的平方相减，避免了开方 `sqrt`！）
    *   把所有雷达的残差加起来：`total_cost += e * e;` （这就是 $e^T e$）
    返回 `total_cost`。
3.  **测试三组猜测**：
    *   猜测一（离谱）：`X1 = [8.0, 8.0]`
    *   猜测二（接近）：`X2 = [3.2, 4.1]`
    *   猜测三（真值）：`X3 = [3.0, 4.0]`
4.  **输出**：打印这三个猜测算出来的 `total_cost`，观察越接近真值，代价是不是越小。
