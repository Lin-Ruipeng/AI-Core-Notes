# API 指南

## 思考模式

1. 思考模式开关

`{"thinking": {"type": "enabled/disabled"}}`

2. 思考强度控制

`{"reasoning_effort": "high/max"}`

思考模式不支持很多参数，思考模式下，思维链通过 reasning_content 参数返回，与 content 同级。
在后续的轮次的拼接中可以选择性返回思考内容。
- 如果模型没有工具调用，则思维链无需参与上下文拼接！
- 如果模型进行了工具调用，则思维链需要参与上下文拼接！

## “Function Calling” 或 “Tool Use”

简单来说，就是用 json 数据包告诉 LLM，我有什么样的函数可以调用，这个函数是干什么的，需要什么参数，每个参数又是什么含义，然后让 LLM 决定是否需要调用工具，如果要调用，那么开发者解包，得到 json，LLM 会把参数填好，开发者用命令行运行指定的函数就可以了，最后把函数的返回结果写入到上下文中，然后 LLM 继续读取上下文得到运行结果，然后 LLM 总结输出，用户得到结果。
